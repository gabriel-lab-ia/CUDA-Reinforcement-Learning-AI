# Benchmark Methodology

## Definition

A benchmark in this repository is a reproducible command-line run that records
configuration, hardware and software metadata, per-run metrics, aggregate
statistics, and a Markdown comparison report. A smoke benchmark validates that
the pipeline works. A formal benchmark requires at least 10 seeds and must not
be inferred from a single execution.

## Warm-up and synchronization

Microbenchmarks execute repeated operations. CUDA backends must synchronize
before interpreting timings. The current smoke implementation synchronizes after
PyTorch CUDA microbenchmarks. Native CUDA timing is listed as in development
until an RL kernel with CUDA events is added.

## Seeds

Smoke benchmark configs use five seeds by default. Formal reports require at
least ten seeds through schema validation.

## Confidence intervals

Aggregate confidence intervals use bootstrap resampling of the mean. The method
is intentionally documented in `src/cuda_rl/benchmarks/statistics.py` and is
used only when there are executed values to summarize.

## Interpretation

Do not treat smoke results as performance claims. Reports generated by the
runner explicitly state this limitation. Speedup claims require completed CPU,
PyTorch CUDA, and native CUDA measurements with numerical correctness checks.
